gitのstageに上がっている変更をレビューしてコミットしてください。

以下を実施してください:
1. `git status`と`git diff --cached`でステージされた変更を確認
2. 変更内容を丁寧にレビュー
   - コーディング規約違反
   - 潜在的なバグ
   - セキュリティ上の問題
   - パフォーマンス上の問題
   - 可読性の問題
3. 修正が必要な箇所を発見した場合:
   - 問題箇所をリストアップして表示
   - **各修正について1つずつユーザーに確認を取る（AskUserQuestionツールを使用）**
   - ユーザーが承認した場合のみ修正を実施
   - **勝手に編集はしない**
4. レビュー完了後、適切なコミットメッセージを提案
   - 最近のコミット履歴を参考にスタイルを合わせる
   - 変更の「何を」だけでなく「なぜ」を含める
   - 1-2文で簡潔に
5. コミットメッセージをユーザーに確認してもらってからコミット実行

重要な注意事項:
- **修正は必ずユーザーの承認を得てから実施すること**
- 複数の修正箇所がある場合は、1つずつ順番に確認すること
- ステージされた変更がない場合は、その旨を伝える
- コミット前に必ずユーザーの最終確認を取ること

完了後:
- コミットハッシュとメッセージを表示
- `git status`で状態を確認して表示
